Support

Here's the list of tools you will need to get started with NLPCraft:

  • Java SE Runtime Environment (JRE) or OpenJDK ver. 8 or later installed.
  • Latest versions of Git and Maven.

For any questions, feedback or suggestions:

  • Send us a note at  support@nlpcraft.org
  • Post a question at  Stack Overflow using nlpcraft tag
  • If you have a bug or an idea open new issue on  GitHub.

By downloading NLPCraft you are agreeing to license, terms & conditions.

GitHub

If you want to contribute to the project or build it form the source you need to get the full developer snapshot from GitHub: 

            $ git clone https://github.com/vic64/nlpcraft.git
            $ mvn clean package

GitHub vs. ZIP vs. Maven

Use GitHub clone if you want to contribute to the project or build it from source. To build your own models and integrate into your own application it is recommended to download ZIP archive or use build tools and add build dependency.

Maven/Grape/SBT

The easiest way to start using NLPCraft is to add Maven dependency (Grape, SBT, etc.) to your project. NLPCraft project consists of a single module.

Maven Java Grape Groovy Gradle Kotlin SBT Scala 
                    <dependency>
                        <groupId>org.nlpcraft</groupId>
                        <artifactId>nlpcraft</artifactId>
                        <version>{{site.latest_version}}</version>
                    </dependency>
                    
                    @Grab ('org.nlpcraft:nlpcraft:{{site.latest_version}}')
                    
                    dependencies {
                        runtime group: 'org.nlpcraft', name: 'nlpcraft', version: '{{site.latest_version}}'
                    }
                    
                    libraryDependencies += "org.nlpcraft" % "nlpcraft" % "{{site.latest_version}}"

Latest Version

{{site.latest_version}} is the latest version used by default. You can replace it with a desired version.

Optional Packages

There are number of optional package that can be installed and used with NLPCraft.

PostgreSQL

You can install and use PostgreSQL as a system database in the REST server instead of the built-in distributed SQL storage from Apache Ignite that is used by default. Add the following dependency to your project:

Maven Java Grape Groovy Gradle Kotlin SBT Scala 
                    <dependency>
                        <groupId>org.postgresql</groupId>
                        <artifactId>postgresql</artifactId>
                        <version>42.2.5</version>
                    </dependency>
                    
                    @Grab ('org.postgresql:postgresql:42.2.5')
                    
                    dependencies {
                        runtime group: 'org.postgresql', name: 'postgresql', version: '42.2.5'
                    }
                    
                    libraryDependencies += "org.postgresql" % "postgresql" % "42.2.5"

Comments:

  • Make sure to change 42.2.5 version to the latest or required one.
  • Update configuration property server.database.jdbc in file application.conf with required JDBC driver class and JDBC URL.
  • Use scripts from sql/postgres folder to create database and initialize DB schema.
  • Note that you can also download PostgreSQL JDBC driver as a separate JAR file and add it to your project classpath if you are not using, or instead of, build tools.
MySQL

You can install and use MySQL as a system database in the REST server instead of the built-in distributed SQL storage from Apache Ignite that is used by default. Add the following dependency to your project:

Maven Java Grape Groovy Gradle Kotlin SBT Scala

                        <groupId>mysql</groupId>
                        <artifactId>mysql-connector-java</artifactId>
                        <version>8.0.15</version>
                    </dependency>
                    
                    @Grab ('mysql:mysql-connector-java:8.0.15')
                    
                    dependencies {
                        runtime group: 'mysql', name: 'mysql-connector-java', version: '8.0.15'
                    }
                    
                    libraryDependencies += "mysql" % "mysql-connector-java" % "8.0.15"

Comments:

  • Make sure to change 8.0.15 version to the latest or required one.
  • Update configuration property server.database.jdbc in file application.conf with required JDBC driver class and JDBC URL.
  • Use scripts from sql/mysql folder to create database and initialize DB schema.
  • Note that you can also download MySQL JDBC driver as a separate JAR file and add it to your project classpath if you are not using, or instead of, build tools.
StanfordNLP

By default, NLPCraft uses Apache OpenNLP as its base NLP engine for PoS tagging, tokenization, NE/NNE resolution, etc. If desired, you can switch to Stanford CoreNLP engine instead.

Maven Java Grape Groovy Gradle Kotlin SBT Scala 
                    <dependency>
                        <groupId>edu.stanford.nlp</groupId>
                        <artifactId>stanford-corenlp</artifactId>
                        <version>3.9.2</version>
                    </dependency>
                    
                    @Grab ('edu.stanford.nlp:stanford-corenlp:3.9.2')
                    
                    dependencies {
                        runtime group: 'edu.stanford.nlp', name: 'stanford-corenlp', version: '3.9.2'
                    }
                    
                    libraryDependencies += "mysql" % "stanford-corenlp" % "3.9.2"

Comments:

  • Stanford CoreNLP is licensed under GNU General Public License v3 - make sure your usage is compliant with this license.
  • Stanford CoreNLP library must be available on both the REST server and the data probe.
  • Make sure to change 3.9.2 version to the latest or required one.
  • Set system property NLPCRAFT_NLP_ENGINE value to stanford (from default opennlp) on both the REST server and the probe JVMs: -DNLPCRAFT_NLP_ENGINE=stanford
  • Note that you can also download Stanford CoreNLP as a separate JAR file and add it to your project classpath if you are not using, or instead of, build tools.
